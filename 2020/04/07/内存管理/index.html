内存管理

作用域和生命周期

C语言变量的作用域分为：

  • 代码块作用域(代码块是{}之间的一段代码)
  • 函数作用域
  • 文件作用域

局部变量

局部变量也称auto自动变量(auto可写可不写)，一般情况下代码块{}内部定义的变量都是自动变量，特点如下：

  • 在一个函数内定义，只在函数范围内有效
  • 在复合语句中定义，只在复合语句中有效
  • 随着函数调用的结束或复合语句的结束，局部变量的生命周期也结束
  • 如果没有赋初值，内容为随机

作用域：作用范围

  1. 普通局部变量
    • 1、在{}内部定义的变量就是局部变量，不初始化则为随机值
    • 2、只有执行到定于变量的这个语句，系统才会给这个变量分配空间
    • 3、当离开{}，这个非static局部变量自动释放
    • 4、局部变量的作用域在当前的{}，离开此{}，无法使用此变量
    • 5、{}的普通局部变量，加不加auto关键字等价，普通局部变量也叫自动变量
    • 6、C语言auto关键字和C++中auto不一样
    • 7、不同{}中，变量名可以一样
  2. static局部变量
    • 1、在{}内部定义的变量就是静态局部变量
    • 2、static静态局部变量，是在编译阶段就已经分配空间，函数没有调用前，它已经存在
    • 3、当离开{}，这个static静态局部变量不会释放，只有程序结束，static变量才自动释放
    • 4、局部静态变量的作用域在当前的{}，离开此{}，无法使用此变量
    • 5、不同{}中，变量名可以一样
    • 6、如果static局部静态变量不初始化，它的默认值为0
    • 7、只有执行到才会初始化，static静态局部变量初始化语句只会执行一次，但可以赋值多次
    • 8、static变量只能用常量初始化
  3. 普通全局变量(外部链接)
    • 1、在{}外面(函数外面)定义的变量为全局变量
    • 2、只要定义了全局变量，任何地方都能使用此变量
    • 3、如果使用变量时，在前面找不到此全局变量的定义，需要声明才能使用
    • 4、全局变量不初始化，默认赋值为0
    • 5、全局变量只能定义一次，可以声明多次，声明只针对全局变量，不是针对局部变量
    • 6、全局变量在编译阶段已经分配空间(函数没有执行前)，只有在整个程序结束，才自动释放
    • 7、不同文件，全局变量只能定义一次，可以声明多次
  4. static全局变量(内部链接)
    • 1、static全局变量和普通全局变量的区别就是作用域不一样(文件作用域)
    • 2、extern关键字只适用于普通全局变量
    • 3、普通全局变量所有文件都可以使用，前提是需要声明
    • 4、static全局变量只能本文件使用，别的文件不能使用
    • 5、不同文件只能出现一个普通全局变量的定义
    • 6、一个文件可以有一个static全局变量的定义，不同文件间的static全局变量，同名变量也是没有关系的变量
  5. 普通函数和static函数的区别(文件作用域)
    • 1、所有文件只能有一次普通函数的定义
    • 2、一个文件可以有一个static函数的定义
    • 3、普通函数所有文件都能调用，前提是使用前声明
    • 4、static函数只能在定义所在的文件中使用

总结：

  • 普通变量：
    • 生命周期是在{}结束时候销毁，
  • static静态变量：
    • 整个程序结束后销毁
  • 全局变量：
    • 所有文件，只能定义一次，别的文件只要extern声明就可以使用
  • static全局变量：
    • 每个文件自能定义一个，跨文件同名也是两个不同变量，不能相互使用

普通局部变量和static静态局部变量区别：

  • 内存分配和释放：
    • a) 普通局部变量只有执行到定义的变量才分配空间
    • b) static局部变量在编译阶段(函数还没执行)，变量空间已经分配
    • c) 普通局部变量离开作用域{}，自动释放
    • d) static局部变量只有在整个程序结束才自动释放
  • 初始化：
    • a) 普通局部变量不初始化，值为随机数
    • b) static局部变量不初始化，值为0
    • c) static局部变量初始化语句只有第一次执行有效
    • d) static局部变量只能用常量初始化

普通全局变量的定义和声明：

  • 1、定义一个全局变量，建议初始化
    int a = 10;
  • 2、如果声明一个全局变量，建议加extern
    extern int a;

内存分区

        C代码经过预处理、编译、汇编、链接四个步骤后生成一个可执行程序
        在程序没有执行前，有几个内存分区已经确定，虽然分区确定，但是没有加载内存，程序只有运行时才加载内存
程序运行前内存大致分区：

  • text(代码区)：只读，函数
  • data：初始化的数据，全局变量，static静态变量，文字常量区(只读)
  • bss：没有初始化的数据，全局变量，static静态变量

当程序运行时，加载内存，首先根据前面确定的内存分区(text、data、bss)先加载，再额外加载两个区，栈区、堆区
运行时内存大致分区

  • text(代码区)：只读，主要是函数
  • data：初始化的数据，全局变量，static静态变量，文字常量区(只读)
  • bss：没有初始化的数据，全局变量，static静态变量
  • stack(栈区)：普通局部变量，自动管理内存,先进后出的特点
  • heap(堆区)：手动申请空间，手动释放，整个程序结束，系统也会回收，如果没又手动释放，程序没有结束，这个堆空间不会自动释放

Linux 中可以使用ulimit -a查看栈空间大小

堆区内存分配和释放

  • 1、动态分配空间，如果程序没有结束，不会自动释放
  • 2、一般用完，需要人为释放free(p)
  • 3、free(p)不是释放p变量，释放p所指向的内存
  • 4、同一块堆区内存只能释放一次
  • 5、所谓的释放不是指内存消失，指这块内存用户不能再使用，系统回收，用户再使用，就是操作非法内存

内存泄漏动态分配了空间，不释放
内存污染非法使用内存
