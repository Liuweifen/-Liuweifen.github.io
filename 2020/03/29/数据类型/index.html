    数据类型
    发表于 2020-03-29| 更新于 2020-03-31
    阅读量:

    标识符命名规范：

    1. 字母、数字、下划线组合，开头不能使用数字
    2. 标识符不能是关键字
    3. 标识符区分大小写

    关键字

    C的关键字一共有32个

    • 数据类型关键字（12个）
      char，short，int，float，double，unsigned，signed，struct，union，enum，void
    • 控制语句关键字（12个）
      if，else，switch，case，default，for，do，while，break，continue，goto，return
    • 存储类关键字（5个）
      auto，extern，register，static，const
    • 其他关键字（3个）
      sizeof，typedef，volatile

    数据类型

            数据类型的作用：编译器预算对象（变量）分配的内存空间大小。

    常量

    • 在程序运行过程中，其值不能够被改变
    • 一般出现在表达式或者赋值语句中
    • 例如：
      整型： 100，-1000，0
      实型： 3.1415926，0.5，-1.23
      字符型常量： ‘a’，‘B’，‘1’，‘\n’
      字符串常量：“123456”，“hello world”

    变量

    • 在程序运行过程中，其值可以改变
    • 变量在使用前必须先定义，定义变量前必须有相应的数据类型

      标识符命名规则：

    • 标识符不能是关键字
    • 标识符只能由字母、数字、下划线组成
    • 标识符中字母区分大小写

      变量特点：

    • 变量在编译时为其分配相应的内存空间
    • 可以通过其名字和地址访问相应内存
    • 同一个{}内部，不能出现2个同名变量
    • 变量的初始化和变量的赋值

    声明和定义的区别

    • 声明变量不需要建立存储空间，如：extern int a;
    • 定义变量需要建立存储空间，如：int b；

      定义是声明的一个特例，一般情况下把建立存储空间的声明称为“定义”，而把不需要建立存储空间的声明称之为“声明”。

    • int b它既是声明，同时又是定义
    • 对于extern b来说它只是声明不是定义
    1. #开头的语句是预处理语句，无需分号结束
    2. 定义一个宏定义的标识符MAX，它代表了100，MAX它是常量
    3. 宏定义以后的代码都可使用MAX
    4. 在后面出现的MAX，预处理都会替换为100
    5. const是一个关键字，作用是修饰一个变量为只读，不允许修改

    案例：

    二进制

          二进制是计算机中广泛的数制，二进制数据采用0和1来表示。它的基数为2，进位规则是“逢二进一”，借位规则是“借一当二”。
          当前的计算机系统使用的基本上是二进制系统，数据在计算机中主要是以补码形式存储的。

    术语和含义

    • bit(比特)：一个二进制代表一位，一个位只能表示0和1两种状态。
    • Byte(字节)：一个字节为8位，计算机中存储的最小单位是字节。数据存储是习惯以“字节”（Byte）为单位。
    • WORD(双字节)：2个字节，16位。
    • DWORD：两个WORD，4个字节，32位。
    • 1b：1bit，1位
    • 1B：1Byte，1字节，8位
    • 1k，1K：1024
    • 1M(1兆)：1024K，1024*1024
    • 1G：1024M
    • 1T：1024G
    • 1Kb(千位)：1024bit，1024位
    • 1KB(千字节)：1024Byte，1024字节
    • 1Mb(兆位)：1024Kb = 1024 * 1024bit
    • 1MB(兆字节)：1024KB = 1024 * 1024Byte

    八进制转二进制

          八进制的数和二进制数可以按位对应(八进制一位对应二进制三位)，因此常用在计算机语言中。

    • 010 111 101 010 101 010 010 101 （二进制）

    • 2 7 5 2 5 2 2 5 （八进制）

    • 1 2 4 5 6 7 （八进制）

    • 001 010 100 101 110 111 （二进制）

    十六进制转二进制

          八进制的数和二进制数可以按位对应(八进制一位对应二进制四位)，因此常用在计算机语言中。

    • 0101 1110 1010 1010 1001 0101 （二进制）

    • 5 e a a 9 5 （十六进制）

    • 9 a 8 f 7 （十六进制）

    • 1001 1010 1000 1111 0111 （二进制）

    八进制转十六进制借助二进制作为中介

    C语言如何表示相应进制数

    • 二进制：C语言不能直接书写二进制数
    • 八进制：以数字0开头，如0123
    • 十进制：正常数字1-9开头，如123
    • 十六进制：以0x开头，如0x123

    计算机内存数值存储方式

          存储1字节（8位）大小的数字（char）

    原码（用户角度）：原始二进制

    1. 用户的数字分为正负数，符号位的存储

    2. 最高位为符号位：0代表正数，1代表负数

    3. 1000 00001 左边是最高位，右边是低位

      • +1：0000 0001
      • -1：1000 0001
      • +0：0000 0000
      • -0：1000 0000

    源码存储导致两个问题：

    1. 0有两种存储方式+0、-0
    2. 正数和负数相加，结果不正确（计算机只会做加法运算不会做减法运算 1-1 == 1 + (-1) ）

    原码计算：

    • +1: 0000 0001
    • -1: 1000 0001
    • =: 1000 0010 = -2

    反码（为了算补码）

    1. 正数的原码和反码是一样的
    2. 先求原码
    3. 在原码基础上，符号位不变，其它位置取反（0变1，1变0）
    • +1： 0000 0001 –取反–> 0000 0001
    • -1： 1000 0001 –取反–> 1111 1110
    • +0： 0000 0000 –取反–> 0000 0000
    • -0： 1000 0000 –取反–> 1111 1111

    源码存储导致一个问题：

    1. 0有两种存储方式+0、-0

    反码计算：

    • +1: 0000 0001
    • -1: 1111 1110
    •  =: 1111 1111 = -0

    补码（计算机存储方式以补码存储：为了解决负数的存储）

    1. 正数的原码、反码和补码都是一样的
    2. 对于负数，其补码是它的反码+1
    • +1： 0000 0001 –补码–> 0000 0001
    • -1： 1111 1110 –补码–> 1111 1111
    • +0： 0000 0000 –补码–> 0000 0000
    • -0： 1111 1111 –补码–> 10000 0000（最高位丢弃） = 0000 0000

    补码 计算：

    • +1: 0000 0001
    • -1: 1111 1111
    •  =: 10000 0000（最高位丢弃） = 0

    原码求补码：

    1. 最高位符号位，其它位就是二进制（原码）
    2. 在1条件基础上，符号位不变，其它位取反
    3. 补码在2条件基础上+1

    补码求原码

    1. 求补码的反码，符号位不变，其它位取反
    2. 原码在2条件的基础上+1

    十进制，站在用户角度考虑（原码）

    二进制、八进制、十六进制，站在计算机角度（补码）

    有符号和无符号的区别

    • 有符号（打印方式%d），最高为是符号位，如果是1代表负数，如果是0代表正数
    • 无符号（打印方式%u），最高位不是符号位，是数的一部分，无符号不可能是负数

    signed int a = -10;
    unsigned int b = 10;

    数据类型范围

    • char：1个字节 有符号范围：
      • 正数：0000 0000 ~ 0111 1111 0 ~ 127
      • 负数：1000 0000 ~ 1111 1111 -0（-128）~ -127
    • short：2个字节 有符号范围： -32768 ~ 32767（-2^15 ~ 2^15 - 1）
    • int：4个字节 有符号范围： -2147483648 ~ 2147483647（-2^31 ~ 2^31 - 1）
    • long：4个字节 有符号范围： -2147483648 ~ 2147483647（-2^31 ~ 2^31 - 1）
    • char：1个字节 无符号范围： 0000 0000 ~ 1111 1111 0 ~ 255
    • short：2个字节 无符号范围： 0 ~ 65535（2^16 - 1）
    • int：4个字节 无符号范围： 0 ~ 4294967295（2^32 - 1）
    • long：4个字节 无符号范围： 0 ~ 4294967295（2^32 - 1）

    赋值或者运算，不要越界

    • char ch = 127 + 2 = -127
    • unsigned char b = 255 + 2 = 1

    sizeof关键字

    • sizeof不是函数，所以不用包含任何头文件，它是用来计算数据类型大小，单位字节。
    • sizeof的返回值为size_t
    • size_t类型在32位操作系统下是unsigned int，是一个无符号的整数

    字符类型

    1. 内存中没有字符，只有数字
    2. 一个数字，对应的一个字符，这种规则就是ASCII
    3. 使用字符或者数字给字符变量赋值是等级
    4. 字符类型本质上就是1个字节大小的整形
    5. 大小转小写+32，小写转大写-32
    6. 字符，原则上‘’内部只有一个字节，转义字符除外
    7. 转义字符由反斜杠’'组成多个字符

    整型

    输出格式控制

    • %d：十进制
    • %o：八进制
    • %x、%X：小写十六进制、大写十六进制
    • %u：无符号十进制
    • %hd：short类型
    • %l：long类型
    • %ll：longlong类型
    • %hu：unsigned short类型
    • %lu：unsigned long 类型
    • %llu:unsinged long long 类型

    类型：short、int、long、long long

    • shotr短整型：2个字节
    • int整型：4个字节
    • long长整型：windows、Linux(32) 4字节 Linux(64) 8字节
    • long long：8字节

    实型（浮点型）

    主要分为两种：单精度浮点型float，双精度浮点型double，float存储可能有小尾巴不准确

    • float： 4字节 7位有效数字
    • double： 8字节 15-16位有效数字

    类型限定符

    • extern：声明一个变量，没有建立存储空间（extern int a）
    • const：定义一个常量，常量值不能修改（const int a = 10）
    • volatile：防止编译器优化代码
    • register：定义寄存器变量，提高效率。register是建议型指令，而不是命令型指令，如果CPU有空闲寄存器，那么register就生效，没有空闲就无效

    字符串的格式化输出和输入

    字符与字符串常量的不同：

    • ‘a’：为字符常量，内存存储为: |’a’|
    • “a”：为字符串常量，内存存储为: |’a’||’\0’|
    • 每个字符串结尾，编译器会自动添加一个结束标志位’\0’，即”a”包含两个字符’a’和’\0’
