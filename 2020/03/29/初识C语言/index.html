发表于 2020-03-29| 更新于 2020-04-01

了解C语言

  • 使用某个函数前，包含相应的头文件（#include “aaa.h”）
  • Linux可以使用man手册查看函数的使用
  • 头文件类似于菜单，头文件包含了函数的声明，比如菜单的菜名
  • <>通过包含系统的头文件（标准的头文件），””包含自定义的头文件

最简单C语言程序结构

  1. C语言由函数组成，有且只有一个主函数main()
  2. 程序运行，先从main函数开始，该函数由系统自动调用，无需人为调用
  3. 函数结束时，return 0程序正常结束
  4. 函数的调用必须放在{}内
  5. C语言源代码文件是普通文件，但是扩展名必须是.c
  6. 每个可执行语句都是以”;”结尾，所有#开头的预编译指令不需要分号结尾

注释 不是有效的代码块

  • 行注释 //后面的代码无效 C++借鉴过来
  • 块注释 /* 这里面的代码无效 */ C语言原生

system函数的使用

#include <stdlib.h>
int system(const char *command);
功能：在已经运行的程序中执行另一个外部程序
参数：外部可执行程序名字
返回值：不同系统返回值不一样

字符编码

1) Windows默认支持的中文编码为GBK，GB2312，ANSI
2) Linux默认支持的中文编码为UTF-8(Unicode)

C语言编译步骤

C代码编译可执行程序经过4个步骤：

  1. 预处理：宏定义展开、头文件展开、条件编译等，同时将注释代码删除，这里没有进行语法检查
  2. 编译：检查语法，将预处理后的文件编译生成汇编文件
  3. 汇编：将汇编文件生成目标文件（二进制文件）
  4. 链接：C语言写的程序是需要依赖各种库的，所以编译之后还需要把库链接到最终的可执行程序中去

分布编译

ESc — >>> 生成 iso 文件

  • 预处理： gcc -E FileName.c -o FileName.i
  • 编 译： gcc -S FileName.i -o FileName.s
  • 汇 编： gcc -c FileName.s -o FileName.o
  • 链 接： gcc FileName.o -o FileName
  • 一次性编译：gcc FileName.c -o FileName(同样经历以上四个步骤)

CPU内部结构与寄存器

32位与64位系统区别：

  • 寄存器是CPU内部最基本的存储单元
  • CPU对外是通过总线（地址、控制、数据）来和外部设备交互的，总线的宽度是8位，同时CPU的寄存器也是8位，那么这个CPU就叫8位CPU
  • 如果总线是32位，寄存器也是32位的，那么这个CPU就是32位CPU
  • 有一种CPU内部的寄存器是32位的，但总线是16位，准32位CPU
  • 所有的64位CPU兼容32位的指令，32位要兼容16位的指令，所以在64的CPU上可以识别32位的指令
  • 在64位的CPU架构上运行64位的软件操作系统，那么这个系统是64位
  • 在64位的CPU架构上运行32位的软件操作系统，那么这个系统是32位
  • 64位的软件不能运行在32位CPU上

运算原理：

  1. 如果需要运输，不能在内存中直接运算
  2. 如果需要运算，把内存中的数据加载到寄存器，再运算
  3. 把运算结果从寄存器移回内存中
  4. 内存通过缓存再加载到寄存器效率会提高
